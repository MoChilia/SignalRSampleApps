SSE Token Expiry Simulation

Purpose: Demonstrates what happens when an access token expires during an active SSE (Server-Sent Events) connection to Azure SignalR Service.

Scenario A (Static Token — BUG): Token is fetched once and reused forever. When it expires, the SSE stream is closed and all reconnect attempts fail with 401.
Scenario B (Dynamic Token — FIX): accessTokenFactory fetches a fresh token on each reconnect. The SSE stream is briefly interrupted but reconnection succeeds automatically.
